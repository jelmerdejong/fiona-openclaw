### Exec Policy Saga (09:00–12:25 UTC)

**What we tried (and didn't work):**
1. `tools.exec.ask: "off"` — only controls approval prompts, not the allowlist
2. `tools.profile: "coding"` → `"full"` → removed entirely — profile was gone but allowlist persisted
3. Cleared `exec-approvals.json` allowlist + set `defaults.security: "full"` in approvals file — still blocked
4. Multiple gateway restarts and /reset — new sessions still inherited the block

**The actual fix:**
- **`tools.exec.security: "full"`** — the real gatekeeper field in openclaw.json
- Was defaulting to "allowlist" mode, enforcing command restrictions regardless of `ask` or approvals
- Protected config path — only settable via CLI (`openclaw config set tools.exec.security full`)
- After setting + gateway restart at ~12:25 UTC: exec fully unrestricted ✅

**Key lesson:** Three separate layers control exec: `tools.exec.security` (allowlist vs full), `tools.exec.ask` (approval prompts), and `exec-approvals.json` (per-command grants). The security field is the master switch.
